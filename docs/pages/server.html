Server

Server is usually your first module in Corpus. It should be instantiated before any Route, Middleware, or Controller object is created. This is because Server's constructor creates and registers the global routing for your app. It is part of the C (Corpus) module which is the main module that holds all necessary parts of an HTTP server.

Public Methods

.listen

Expose a port and start listening. This is pretty straightforward. The method takes a required port and an optional hostname arguments. This is an async method to allow async operations in handleBeforeListen however, there is no return value.
Example: 

import { C } from "@ozanarslan/corpus";

const server = new C.Server();
// Since this is async, you might want to satisfy 
// the linter by using void or await.
void server.listen(3000, "0.0.0.0");

.close

The counterpart of listen is close. This method stops the underlying serve function to prevent new connections from being accepted. Takes closeActiveConnections boolean argument. This is also async to allow async operations in handleBeforeClose.
Note: By default, closeActiveConnections is true. Bun's default is false.
Example: 

void server.listen(3000, "0.0.0.0");
void server.close();
